# Fury Changelog

按时间倒序维护，推荐每个版本固定填写三个分类：`新增`、`优化`、`修复`。

## v2.9 (2026-03-22)

### 新增
- 时间线已施放图标新增结果摘要显示：可直接展示命中伤害、未命中结果、破甲层数与战吼影响人数。

### 优化
- 多目标伤害技能现在会在时间线上按单次施法汇总总伤害，不再只显示技能名。
- 暴击、普通命中、擦伤、格挡与未命中结果的时间线文字颜色已区分，战斗反馈更直观。

### 修复
- 修复时间线已施放图标在连续命中、多目标与后续 Aura 回写场景下无法稳定反映真实结果的问题。

## v2.8 (2026-03-21)

### 新增
- 无

### 优化
- 重构底层的状态扫描机制，大幅降低单帧计算开销，彻底消除复杂战斗环境下的游戏卡顿与掉帧现象。

### 修复
- 彻底解决在国服或硬核高负载环境下（同名目标或光环过多时）引发单帧指令数超限（`script ran too long`），导致插件报错甚至直接停跳的严重硬件限流问题。

## v2.7 (2026-03-21)

### 新增
- 无

### 优化
- 大幅优化决策树在高频战斗事件下的计算性能，降低 CPU 占用并减少掉帧。

### 修复
- 彻底修复嗜血、盾猛等天赋核心技能在部分客户端被误判为未学习，导致完全不提示的问题。
- 修复非编辑模式下界面背景板无法自动隐藏的问题。
- 修复极端情况下由于局部变量超限导致的插件严重崩溃及推荐图标彻底消失的问题。

## v2.6 (2026-03-21)

### 新增
- Debug 面板打开时，`BT` / `OP` 关键决策链会同步打印到聊天，便于现场定位主技能为何未进入推荐树。

### 优化
- 主技能与 `Dump` 的关系重新收口：当 `BT/WW/EX/OP` 进入 `ready-soon` 窗口时，主提示会优先为主技能预留。
- 技能解析底层改为法术书驱动，`token` 与真实 `spell id` / `spell name` 的映射会跟随天赋、学技能和技能页变化自动刷新。

### 修复
- 修复 `Bloodthirst` 等天赋技能在部分客户端里被误判为未学习，导致直接被排除出决策池的问题。
- 修复点出天赋后技能 `spell id` 变化无法被旧静态映射识别，导致主提示、时间线与施法回写失真的问题。

## v2.5 (2026-03-21)

### 新增
- Debug 面板打开时，`Overpower` 异常推荐会同步打印到聊天，便于现场定位姿态、窗口和目标状态。

### 优化
- Debug 面板展示字段收敛为真实的 `展示源` 与 `展示GCD`，不再混用旧兼容口径。
- `GCD`、天赋点数与玩家 Aura 读取补充 Classic Era 兼容回退，降低客户端返回差异导致的假死数据。

### 修复
- 修复 `Overpower` 会从 `wait prediction` / `ready-soon lock` 兼容路径错误漏到 HUD 主提示的问题。
- 修复主提示区仍可能从旧 `displayNextSkill` 回退路径残留旧技能的问题。
- 修复 `Death Wish` 在部分客户端落入 debuff 栏时 Debug 面板 Buff 状态无法识别的问题。

## v2.4 (2026-03-20)

### 新增
- HUD 新增独立编辑模式，开启后可拖拽位置，关闭后恢复点击穿透。

### 优化
- HUD 交互入口收敛为单一编辑模式，移除了多余的锁定开关与对应命令说明。
- 时间线 marker 改为按需扩容并提高事件上限，避免高频施放时旧图标在时间窗内被提前挤掉。

### 修复
- 修复 HUD 透明区域仍会拦截鼠标点击的问题。
- 修复非编辑模式下背景板仍可见的问题，现在仅在编辑模式显示背景与边框。

## v2.3 (2026-03-20)

### 新增
- 主提示图标与时间线整合到同一 HUD 容器，推荐动作在 `queued` 或成功施放后会平滑并入时间线。

### 优化
- 时间线图标尺寸按主图标大小动态放大，轨道与主提示的视觉比例重新收口。
- 设置页、帮助命令与 README 重新复核，移除了已弃用的脱战图标开关入口。

### 修复
- 修复推荐被替换时缺少退出动画的问题，未 `queued` / 未施放的旧推荐现在会向左位移并淡出。
- 修复时间线透明度被父级框体提前压暗的问题，现在只在最后 `1s` 按单图标线性淡出。

## v2.2 (2026-03-20)

### 新增
- 战斗姿态下新增 `Overpower`（压制）决策与键位提示，在目标刚躲闪后的有效窗口内参与推荐。

### 优化
- 压制窗口监听收紧为仅在战斗姿态下记录目标躲闪，减少无效采集与额外开销。
- 破甲低血量阈值恢复为额外硬拦截条件：目标血量低于阈值且已有层数时不再提示，默认阈值调整为 `50000`。

### 修复
- 修复破甲设置页的 HP 阈值显示未做千分位格式化的问题。

## v2.0 (2026-03-20)

### 修复

## v1.10 (2026-03-19)

### 新增

- 修复候选排序未区分 `passed` 状态的问题，避免 rejected 技能压过真实可执行候选。
- 修复 Execute 描述模型长期缓存不失效的问题，登录、升级和技能变化后会重新解析。

## v1.8 (2026-03-18)

### 新增
- 新增断筋骗乱舞（Hamstring-Flurry）决策分支，在满足保护窗与怒气安全条件时参与主循环候选。
- 新增非满级/未学技能的降级决策矩阵，候选会按当前等级与已学最高 rank 自动适配。
- 设置面板新增 `介绍` 与 `更新` 分页，可在游戏内直接查看作者信息、功能简介与版本记录。

### 优化
- 技能可用性判定统一按“已学最高 rank”解析，降低多 rank 场景下的判定偏差。
- DPS 候选评分改为“真实技能数据驱动”的 rank 收益缩放（按当前已学 rank 相对满级 rank 归一化），低等级阶段不再仅依赖线性等级因子。
- 设置页所有分页与 Debug 面板完成统一排版微调，提升小窗口下文字可读性与层级清晰度。

### 修复
- 修复部分技能在未学习或 rank 不完整时仍被纳入候选的问题，并补充可解释拒绝原因。
